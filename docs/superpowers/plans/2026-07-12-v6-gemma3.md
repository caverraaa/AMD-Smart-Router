# v6 Gemma-3-4B Local Generalist Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Architecture:** Model file + per-category caps swap inside the existing validated machinery; routing table rewritten from fresh judged measurements under the new greedy policy; heavy-slice check before ship. Spec: `docs/superpowers/specs/2026-07-12-v6-gemma3-local-generalist-design.md`.

**Tech Stack:** unchanged (llama-cpp-python 0.3.34, pytest, existing judge harness).

## Global Constraints

- Timeline gates: spike GO/NO-GO by T+1h; golden + routing table by T+2.5h; shipped by T+4h; else ABORT (pointer stays on v4-hybrid).
- 4 GB RAM / 2 vCPU: spike must confirm load ≤40 s, worst answer ≤25 s at max_tokens=320, RSS headroom.
- Ship floor: judged ≥85% on the container output AND projected eval tokens well below 8,262.
- Greedy lane policy: grant by tokens-saved-per-golden-point-lost while projected golden global ≥85%; never grant a lane ≤6/12; revoke any granted lane that goes 0/2 on heavy slice with cloud 2/2.
- All Phase-1/2 invariants preserved (fallbacks, snapshots, temp-0, probe, exit-0).
